Вот пример задания для тренировки работы с GitHub:

---

**Задание:** Создайте репозиторий на GitHub и выполните следующие шаги:

1. **Создание локального репозитория:**
   - Создайте новый проект на своем компьютере.
   - Инициализируйте в нем Git командой `git init`.
   
2. **Добавление файлов:**
   - Добавьте несколько файлов в ваш проект. Например, создайте файл `README.md` с описанием проекта и любой другой файл, который вы хотите включить в репозиторий.
   - Используйте команду `git add .`, чтобы добавить все файлы в индекс.

3. **Коммит изменений:**
   - Сделайте первый коммит с сообщением "Initial commit". Для этого используйте команду `git commit -m "Initial commit"`.

4. **Публикация на GitHub:**
   - Перейдите на сайт GitHub.com и создайте новый пустой репозиторий с тем же именем, которое у вас на локальной машине.
   - Следуйте инструкциям, которые будут показаны после создания репозитория, чтобы связать ваш локальный репозиторий с удаленным.
   - Отправьте изменения на GitHub с помощью команды `git push origin main`.

5. **Работа с ветками:**
   - Создайте новую ветку с названием `feature` и переключитесь на нее командой `git checkout -b feature`.
   - Внесите небольшие изменения в какой-нибудь файл и сделайте коммит этих изменений.
   - Переключитесь обратно на основную ветку (`main`) командой `git checkout main`.

6. **Объединение веток:**
   - Объедините вашу ветку `feature` с основной веткой командой `git merge feature`.

7. **Решение конфликтов:**
   - Если возникнет конфликт при слиянии, решите его вручную, отредактировав соответствующий файл, а затем добавьте его в индекс и завершите слияние командой `git add filename && git commit -m 'Resolved conflict'`.

8. **Отправка изменений на GitHub:**
   - После всех изменений отправьте обновленную версию основного репозитория на GitHub командой `git push`.

9. **Просмотр истории:**
   - Просмотрите историю коммитов вашего репозитория с помощью команд `git log` и `gitk`.

10. **Взаимодействие с другими пользователями:**
    - Пригласите кого-то из друзей или коллег к вашему проекту через интерфейс GitHub, предоставив им доступ к репозиторию.
    - Пусть они внесут свои изменения и сделают pull request. Обсудите предложенные изменения и примите их или отклоните.

11. **Использование GitHub Issues:**
    - Создайте issue на GitHub для отслеживания какого-нибудь бага или задачи.
    - Решите эту задачу, создав отдельную ветку, сделав необходимые изменения и закрыв issue через коммит.

12. **Удаленное взаимодействие:**
    - Клонируйте свой репозиторий на другом устройстве и попробуйте внести изменения оттуда, отправляя их на GitHub и синхронизируя с основным устройством.

13. **Заключение:**
    - Проверьте, что ваш репозиторий выглядит так, как задумано, и все ваши изменения корректно отображаются на GitHub.

---

Это задание охватывает базовые операции с Git и GitHub: создание репозитория, работа с файлами, коммитами, ветвями, слиянием, разрешением конфликтов, взаимодействием с другими участниками и использованием возможностей платформы GitHub.
